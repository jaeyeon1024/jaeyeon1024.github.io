파라미터 기본 정보

temperature
범위: 0 ~ 2.0

top_k
확률 상위 k개 토큰 후보 선택
먼저 적용됨
top_p
누적 확률 p 이내 토큰 선택
top_k 이후 적용됨

실험 결과

EXP · 01  |  temperature=2.0, top_k=1 Gemini 
generation_config = {
  "temperature": 2.0,
  "top_k": 1,
  "max_output_tokens": 2048,
}

질문: 37 * 25 = ?

동일한 질문을 두 번 실행했을 때, 같은 질문에 대해 다른 풀이 방식으로 응답이 생성되었다.

— v1: 분배 법칙(30+7)×25 방식 사용

— v2: 25=100/4 치환 방식, 세로곱셈, 분배 법칙 등 세 가지 방식 제시

→ 같은 질문에도 temperature에 따라 여러 응답이 생성됨
EXP · 02  |  temperature=2.0, top_k=1 Gemini 
generation_config = {
  "temperature": 2.0,
  "top_k": 1,
  "max_output_tokens": 2048,
}

질문: 판타지 소설의 첫 문장 선택지를 줘

temperature=2.0 / top_k=1 조합으로 두 번 실행한 결과, 두 번 모두 동일한 10개의 문장 선택지가 출력되었다.

— top_k=1 이면 매 토큰마다 가장 확률 높은 단 하나의 후보만 선택되므로, temperature가 높아도 선택의 여지가 없어 출력이 고정된다.

→ top_k와 top_p가 낮으면 temperature에 무관하게 고정된 답을 출력함
EXP · 03  |  temperature=0.2, top_k=10, top_p=0.9 Gemini 
generation_config = {
  "temperature": 0.2,
  "top_p": 0.9,
  "top_k": 10,
  "max_output_tokens": 512,
}

질문: 판타지 소설의 첫 문장 선택지를 줘

동일한 질문을 두 번 실행했을 때 서로 다른 선택지가 출력되었다.

— 1회차: 5개의 문장 (각 문장의 주제와 스타일이 명확히 제시됨)

— 2회차: 5개의 다른 문장 (신비로운 분위기, 주인공의 특별함 등 다른 방향성)

temperature가 낮음에도 top_k=10, top_p=0.9 로 후보 폭이 넓어 응답이 달라졌다.

→ temperature가 낮아도 top_k와 top_p가 높으면 다양한 답변이 생성됨

결론

top_k와 top_p가 높을수록 다양한 답변을 생성할 수 있고, 낮을수록 정형화된 고정 답변을 출력한다.

또한 top_k와 top_p가 모두 어느 정도 높을 때 비로소 temperature가 출력 다양성에 영향을 주는 것으로 추측할 수 있다.

즉, top_k=1처럼 후보가 단 하나로 고정된 상태에서는 temperature 값이 아무리 높아도 매 토큰마다 선택의 여지가 없으므로 결과가 항상 동일하게 수렴한다.
